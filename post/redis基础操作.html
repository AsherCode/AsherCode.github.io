redis基础操作

发表于 2018-02-05 | 分类于 redis | 热度: ℃
字数统计: 4,819 | 阅读时长 ≈ 21

老虎不发威，你当我…当我女朋友吧

redis环境搭建,配置,基础命令,持久化,事务,主从复制

优点

  • 基于内存，读写快，键值可以是字符串，列表，散列，集合，有序集合，HyperLogLog
  • 持久化将内存数据保存到硬盘里，方便数据备份和恢复
  • 发布和订阅，消息同时分发多个客户端，构件广播系统
  • 过期键，到期自动删除
  • 事务功能，提供乐观锁，保证处理数据的安全性
  • Lua脚本，在服务器原子地执行多个操作，完成复杂功能，减少客户端和服务器之间的通信往返时间
  • 复制，为指定redis创建一或多个复制品，分担读请求的负载
  • Sentinel,监控服务器状态，进行自动故障转移
  • 集群，创建分布式数据库，每个服务器分别执行一部分读操作和写操作

安装

conf配置

  • 1.daemonsize no 改为yes启用守护进程
  • 2.pidfile /var/file/redis.pid 以守护进程运行时默认写入redis.pid中
  • 3.port 6379 默认监听端口
  • 4.bind 127.0.0.1 绑定的主机地址
  • 5.timeout 300 闲置多长时间后关闭连接，0标识关闭该功能
  • 6.loglevel verbose 指定日志记录级别，debug，verbose，notice，warning默认 verbose
  • 7.logfile stdout 日志记录方式，默认为标准输出，若守护进程方式则日志将发送到/dev/dull
  • 8.databases 16 设置数据库的数量，默认数据库为0，可以使用SELECT 在连接上指定数据库id
  • 9.save 900 1 指定在多长时间内有多少次更新操作，就将数据同步到数据文件，在900s内有一个更改
  • 10.rdbcompression yes 指定存储本地数据是否压缩数据，默认yes，Redis采用LZF压缩，若为了节省cpu时间可关闭
  • 11.dbfilename dump.rdb 指定本地数据库文件名，默认为dump.rdb
  • 12.dir ./ 指定本地数据库存放目录
  • 13.slaveof 设置当本机为slave服务时设置master服务的IP地址及端口，在Redis启动自动master进行数据同步
  • 14.masterauth 当master服务设置密码保护时，slav服务连接master的密码
  • 15.requirepass 123456 设置Redis连接密码，./src/redis-cli -h 127.0.0.1 -p 6379 -a 123456或者连接后auth 123456可以解锁
  • 16.maxclients 128 设置同一时间最大客户端连接数，默认无限制，Redis可以同时打开Redis进程最大文件数，0标识不做限制
  • 17.maxmemory 指定Redis最大内存限制，Redis在启动时把数据加载到内存到最大内存后，尝试清除已到期的key，无法进行写操作。vm机制使key存放内存，value存放到swap区
  • 18.appendonly no 指定是否每次更新操作后进行日志记录，默认异步，不开启则可能断电导致数据丢失
  • 19.appendfilename appendonly.aof 指定更新日志文件名默认appendonly.aof
  • 20.appendfsync everysec 指定更新日志每秒同步一次，no标识操作系统进行数据缓存同步到磁盘，always每次更新操作后手动调用fsync()将数据写到磁盘
  • 21.vm-enabled no 指定是否启用虚拟内存机制，默认值为no，将数据分页存放，冷数据swap到磁盘上，访问多的页面由磁盘自动换出内存中
  • 22.vm-swap-file /tmp/redis.swap 虚拟内存文件路径默认值为/tmp/redis.swap不可多个Redis实例共享
  • 23.vm-max-memeory 0 将所有大于vm-max-memory的数据存入虚拟内存，0时则所有value存在于磁盘
  • 24.vm-page-size 32 Redis swap分成了多个page，一个对象可以保存在多个page上面，但一个page不能被多个对象共享，vm-page-size是要根据存储的数据大小来设定的。若存储很多小对象，page大小最好设置为32或者64bytes，如果存储很大对象，则可以使用更大的page
  • 25.vm-pages 134217728 设置swap文件中的page数量，由于页表放在内存中，磁盘每8个pages将消耗1bytes的内存
  • 26.vm-max-threads 4 设置访问swap文件的线程数，最好不要超过机器的核数，若设置为0，那么所有对swap文件操作是串行的，可能会造成比较长时间的延迟，默认值为4
  • 27.glueoutputbuf yes 设置在向客户端应答时，是否把较小的包含合并为一个包发送，
  • 28.hash-max-zipmap-entries 64 hash-max-zipmap-value 512 指定在超过一定的数量或者最大的元素超过某一临界值
  • 29.activerehashing yes 指定是否激活重置哈希，默认为开启
  • 30.include /path/to/local.conf 指定包含其他的配置文件，可以同一主机上多个Redis实例之间使用同一份配置文件，而同时各个实例又拥有自己的特定配置文件

案例

php_redis

发布订阅

基于快照的持久化rdb

  • 1.redis.conf下snapshoting模块save 120 10标识120秒内key改变10次则自动备份内存数据到本地则自动生成dump.rdb文件。dbfilename dump.rdb标识备份的文件名，dir ./保存的目录，相对于redis-server的位置 ,bgsave调用 rdbSave 函数,手动触发快照持久化
  • 2.flushall后甚至shutdown后迅速刷新rdb文件,cp dump_bk.rdb dump.rdb 将自动恢复数据,save将主动备份刷新rdb文件，bgsave后台异步同时处理前台数据,dbfilename rdb名,dir 标识rdb存放目录,
  • 3.stop-writes-on-bgsave-error yes标识数据不一致将停止写数据
  • 4.rdbcompression 对于存储在磁盘中的快照设置是否进行压缩存储
  • 5.rdbchecksum 存储快照后使用crc64算法进行数据校验
  • 6.自动读取dump.rdb移动到redis安装目录，适合大规模恢复，完整性一致性数据不高
  • 7.在一定间隔时间做备份，若意外down则丢失最后一次快照后的所有修改
  • 8.fork时内存中的数据被克隆了一份，大致2倍的膨胀性需要考虑
  • 9.redis-cli config set save “”停止rdb备份
  • 10.redis-check-dump 恢复rdb文件

基于文件追加方式持久化aof

  • 1.以日志形式记录每个写操作，只许追加文件，重启将根据日志文件内容将写指令从前到后执行一次完成数据的恢复
  • 2.appendonly yes 打开持久化，连接将自动创建日志文件, appendfilename “appendonly.conf” 连接将自动从该文件获取所有写操作包括flushall,appendfsync always 只要存在修改数据则记录到磁盘文件，everysec标识每秒对redis更改计入文件，no标识完全交给操作系统不繁忙时才记录 ,flush shutdown 将自动生成dump.rdb,若aof文件损坏，redis-check-aof –fix appendonly.aof修复才可启动redis
  • 3.appendfsync always同步持久化，立即记录性能较差，everysec异步操作每秒记录，若一秒宕机则数据丢失，No, no-appendfsync-on-rewrite 重写时是否用appendfsync,默认no保证安全性
  • 4.rewrite 避免文件越来越大，新增重写压缩，可命令bgrewriteaof, auto-aof-rwrite-percentage 100, auto-aof-rewrite-min-size 64mb
  • 5.每秒同步，每修改同步，不同步，不过远大于rdb文件，恢复速度慢于rdb，运行效率慢于rdb，每秒同步较好

rdb能够在指定时间间隔对数据进行快照存储
aof记录每次对服务器写操作，重启将调用这些命令恢复原始数据，追加到文件末尾
redis能对aof文件进行后台重写，使得aof文件体积不至于过大
若只希望数据在服务器运行时存在，可以不做持久化
同时开启两种持久化，redis将有限载入aof恢复原始数据，rdb出现问题，只找aof，尽力不用aof备份
建议只在slave持久化rdb，save 900 1
若enable aof，丢失最多2s数据，带来持续io，aof rewrite新数据写到新文件造成阻塞，修改aof重写基础值5G
若不enable aof，仅靠master-slave replication 实现高可用。省去io及rewrite的波动，若master、slave同时倒丢失十几分钟的数据，启动脚本比较两个master/slave的rdb，载入新者
