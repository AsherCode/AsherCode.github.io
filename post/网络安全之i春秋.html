windows 权限分析和密码安全测试

常规用户

System:本地机器上拥有最高权限的用户
Administrator：基本上是本地机器上拥有最高权限的用户
Guest：拥有相对极少的权限，在默认情况被禁用

常规用户组

Administrators：本地机器拥有最高权限
Backup Operators:不如Administrators ,但是差不多
Guests:与USER权限相同
Distributed COM USERS:分布式系统账号，涉及域及域控制器
Network Conguration Operators:windows2003新增用户组，有足够权限去管理网络配置情况
Performance Log Users:windows2003新增用户组，这个用户有权从远程安排性能计数器的日志工作
Performance Monitor Users:windows2003新增用户组，有权从远程监控计算机的运行情况
Power Users:低于Administrators，但高于Guests,不能将添加管理员
Print Operators:不然Administrators,但差不多
Users:本地所有用户账户，低权限的用户组
IIS_WPG:windows2003新增用户组，安装了IIS用来运行和web应用程序的账户将被容纳该用户组

非法文件上传漏洞技术解析及防御

网站前端对上传文件类型限制，可以F12修改点击触发事件，还可以修改事件中添加php，用来添加一句话php后门，后用菜刀连接
<?php @eval($_POST[‘c’]);?>

修改类型名字为php2,php3,php4,php5，绕过文件扩展名过滤

MIME类型绕过漏洞，上传图片，用Burp_Suite_Pro拦截Intercept is on抓到Content-Type类型(image/pjpeg)复制到php一句话木马的拦截(text/plain)的Raw里并修改其Content-Type,绕过MIME检测
jdk1.8，Burp_Suite_Pro_v1.7.32_Loader_Keygen，安装http://www.vuln.cn/8847

  1. 打开注册机：burp-loader-keygen.jar，然后点击run，license text 随便填，然后将生成的license 复制粘贴到打开的burp里，点击next
  2. 点击manual activation 手动激活
  3. 将request 粘贴到activation request ，将自动生成response，再粘贴到burp里最下面的response中，点击下一步
    配置IE浏览器局域网代理服务器：127.0.0.1:8080，在Proxy里的option中配置127.0.0.1:8080的Proxy Listeners
    设置Proxy的Intercept is on 则拦截所有IE访问的地址，并打印raw，Intercept is off则放开

根据文件内容检测，在图片上传时用burp检测拦截后在图片内容乱码最后添加一句话木马，并修改filename后缀为php,再放开提交

空字节截断目录路径检测绕过类上传漏洞，burp拦截后在uploadimg的保存地址后添加1.php .jpg,进入hex二进制格式查看空格对应的20改为00回车，只保留1.php空字节截断

IIS6.0目录路径检测解析绕过上传漏洞，修改路径uploadimg/后添加 1.asp/ 并且修改content-Disposition里的filename为 yijuhua.jpg
<%eval request(“1”)%> iis将/后内容截断，当成注释，不进行解析，直解析到asp，路径是xx.com/uploadimg/1.asp/123123.jpg

IIS6解析缺陷绕过上传漏洞，修改上传路径uploadimg后添加/1.php; 修改content-Disposition里的filename为 yijuhua.jpg
路径是xx.com/uploadimg/1.php;123123.jpg

Apache解析缺陷绕过上传漏洞，修改木马php文件为7z后缀，若apache没有针对解析则以php解析
路径是xx.com/uploadimg/yijuhua.php.7z

htaccess文件上传解析漏洞，上传.htaccess，再将木马改为jpg格式上传
AddType application/x-httpd-php .jpg 是php文件以jpg解析

  • 客户端检测，使用js对上传图片检测，包括文件大小，文件扩展名，文件类型等
  • 服务端检测，对文件大小，文件路径，文件扩展名，文件类型，文件内容检测，对文件重命名
  • 其他限制，服务端上传目录设置不可执行权限

漏洞挖掘

site:baidu.com inurl:url=http:// 在域名baidu.com的子域名中含有url=http://,可能实现url跳转，远程链接访问代理等，往往存在任意url跳转，xss，ssrf，任意文件读取等漏洞
点击继续访问时，链接添加=javaScript:alert(document.domain)//http:// 获取主域名
burpsuite:
1.关注手机app或微信公众号的网页
2.手机电脑局域网，burspsuite设置监听所有来源，手机设置代理后安装Burpsuite证书，开始抓包
3.自动化漏洞挖掘，burp插件python，安装这个插件后自己写python代码处理burpsite代理中的request/response请求。比如取出请求的url及参数，发送到sqlmapapi实现自动化注入漏洞检测
